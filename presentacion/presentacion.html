Explicación del Proyecto

¿Qué hace el sistema?

Es una aplicación de consola en Python 3.10+ que gestiona un dataset de países desde un archivo .csv. Permite al usuario interactuar con los datos a través de un menú.

Funcionalidades Clave

  • Gestión de Datos: Agregar nuevos países y Actualizar población/superficie de existentes.
  • Consultas: Buscar países por nombre (coincidencia parcial o exacta).
  • Filtros: Por Continente, Rango de Población y Rango de Superficie.
  • Ordenamiento: Por Nombre, Población o Superficie (ascendente o descendente).
  • Estadísticas: Calcular promedios, max/min y cantidad de países por continente.

Explicación del Código: Arquitectura

El código se organizó en "capas" de funciones para ser más ordenado y fácil de mantener.

1. Motor Principal (`programa_principal`)

Es el "cerebro" de la app. Muestra el menú principal y usa match-case para decidir qué función llamar según la opción del usuario.

2. Lógica de Negocio (Ej: `AgregarPais`, `FiltrarPaises`)

Contiene las reglas del sistema. Estas funciones saben *qué hacer* con los datos (cómo agregar, cómo filtrar, cómo ordenar).

3. Datos y Utilidades (Ej: `ObtenerPaises`, `PedirNombre`)

Funciones de Datos: (`ObtenerPaises`, `PersistirCsv`) Son las únicas que saben leer y escribir en el archivo .csv.

Funciones de Utilidad: (`PedirNombre`, `PedirCantidad`) Validan la entrada del usuario. Se aseguran de que no se ingresen letras en números o campos vacíos.

Estructuras de Datos Fundamentales

  • Diccionarios: Usados para modelar cada país. Permite un acceso semántico a los datos (ej: `pais["POBLACION"]`).
  • Listas: Usadas para almacenar el dataset completo. Fue la estructura central que contiene la "lista de diccionarios".

Técnicas Clave de Programación

  • `sorted(..., key=...)`: Esencial para ordenar la lista de diccionarios usando un valor interno (como población o nombre).
  • `max(..., key=...)` y `min(..., key=...)`: Permitieron encontrar el país (el diccionario entero) con la mayor o menor población.
  • `diccionario.setdefault()`: Técnica muy eficiente usada para agrupar países por continente en las estadísticas.

Conclusión y Aprendizajes

  • La Modularización fue Esencial: Dividir el código en funciones con una sola responsabilidad (como `PedirNombre`, `PersistirCsv`) fue el aprendizaje más importante. Hizo que probar y corregir errores fuera muchísimo más simple.
  • Listas de Diccionarios: Esta combinación demostró ser increíblemente versátil y potente para manejar datos estructurados en Python.
  • Validación de Entradas: Separar la validación en funciones "ayudantes" (Utilities) es clave para crear un programa robusto que no falle con entradas incorrectas.
  • Trabajo en Equipo: El uso de Git y una comunicación clara fue fundamental para integrar el trabajo de ambos y gestionar las versiones del código.
